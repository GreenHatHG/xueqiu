# 基础浏览器资料目录怎么用

现在不用你自己手动起 `--cdp` 的 Chrome 了。

程序会自己做这几步：

1）先用你给的 `--user-data-dir` 起一个 Chrome
2）如果这里还没登录，你就在这个窗口里手动登录一次
3）后面每抓一个用户，程序都会复制一份新的资料目录
4）再用这份新目录起一个新的 Chrome 去抓

所以这个目录现在的意思很简单：

- 它是“母本”
- 不是所有用户共用到底的那个目录

## 1. 第一次怎么跑

先准备一个用户文件，比如：

```text
9650668145
```

然后执行：

`uv run xq-crawl --user-list-file data/user_ids.txt --since 2026-03-06`

第一次如果没登录，程序会先弹一个 Chrome。你在里面把雪球登录好就行。

## 2. 想换一份全新的母本

如果你怀疑原来的基础目录已经脏了，最简单就是换个新目录重新登录：

`uv run xq-crawl --user-list-file data/user_ids.txt --since 2026-03-06 --user-data-dir .playwright/user-data-2`

这会把 `.playwright/user-data-2` 当成新的母本目录。

## 3. 每个用户的临时目录在哪

每次正式抓的时候，程序会在这里放临时目录：

`data/browser_profiles/<时间戳>/`

每个用户会有自己单独的一份目录，名字大概像：

`001_9650668145`

规则是：

- 成功了就尽量删掉
- 失败了就保留，日志里会把路径打出来

## 4. 这种做法解决什么

主要就是解决这类问题：

- 前面某个用户把浏览器会话跑脏了
- 后面用户跟着一起怪
- 你删掉整份 `user_data` 以后又好了

现在改成“母本 + 每个用户复制一份新目录”以后，前面用户的脏状态就不容易传到后面。

## 5. 常见情况

### Q1：第一次还是让我登录

正常。第一次就是要把母本目录准备好。

### Q2：后面为什么每个用户都还会起新浏览器

这是故意的。目的就是别让用户和用户之间互相污染。

### Q3：我想只跑一个用户

也还是用 `--user-list-file`。文件里只放一行就行。

### Q4：我怀疑母本又脏了

最省事的办法就是换一个新的 `--user-data-dir`，重新登录一次，再跑。
